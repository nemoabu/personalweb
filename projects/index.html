Study Summary: Muscl3 vs. MuscleWiki

Objective

Evaluate the effectiveness of two anatomy/fitness tools among 20 randomly assigned participants.

Group A MuscleWiki (static website)
Group B Muscl3 (3D interactive model)

Engagement

Muscl3 clearly outperformed MuscleWiki in participant engagement.

  • 3D model kept users focused and immersed
  • Prevented wandering clicks and off-topic exploration
  • More compelling visual experience

Recall & Knowledge Retention

Muscl3 again came out on top.

  • Better recall of finer anatomical details
  • "Recommended by Creators" feature facilitated discovery
  • Enhanced memory retention among all users

Learning Exercises

MuscleWiki Strengths

  • In-depth written explanations
  • Detailed exercise walkthroughs
  • Nuanced technique coverage

Muscl3 Strengths

  • Concise, expandable video content
  • Single-page efficiency

While this guide uses Arch Linux, you can use any Linux distribution. Simply adapt the package installation commands to your distribution's package manager:

pacman -S for Arch apt install for Debian/Ubuntu dnf install for Fedora

Initial Setup

  1. Install your chosen Linux distribution (minimal installation recommended)
  2. Create a non-root user with sudo privileges
  3. Update the system: sudo pacman -Syu
  4. Install essential packages: sudo pacman -S base-devel git openssh tailscale

A minimal installation reduces resource usage and attack surface, perfect for a dedicated server.

How Tailscale Works

  • Mesh Network: Creates a private network connecting all your devices
  • Zero Config: Automatically handles NAT traversal and routing
  • Security: End-to-end encryption between devices
  • Access Control: Granular permissions for device access

Optimization

Disable Unnecessary Services

sudo systemctl disable gdm        # Disable GUI
sudo systemctl disable bluetooth  # Disable Bluetooth
sudo rfkill block wifi          # Disable WiFi

Power Management

  1. Install TLP: sudo pacman -S tlp tlp-rdw
  2. Enable TLP: sudo systemctl enable tlp.service
  3. Configure in /etc/tlp.conf: 
    CPU_SCALING_GOVERNOR_ON_AC=powersave
CPU_SCALING_GOVERNOR_ON_BAT=powersave
WIFI_PWR_ON_AC=off
WIFI_PWR_ON_BAT=off
RUNTIME_PM_ON_AC=auto
RUNTIME_PM_ON_BAT=auto

Tailscale Setup

  1. Start Tailscale service: sudo systemctl start tailscaled
  2. Connect to Tailscale network: sudo tailscale up
  3. Enable auto-start: sudo systemctl enable tailscaled
  4. Configure subnet routing (if needed): sudo tailscale up --advertise-routes=192.168.0.0/24

Git Server Setup

  1. Create git user: sudo useradd -m -r git
  2. Set up SSH keys: 
    sudo mkdir -p /home/git/.ssh
sudo cp ~/.ssh/authorized_keys /home/git/.ssh/
sudo chown -R git:git /home/git/.ssh
  3. Create bare repository: 
    sudo mkdir /srv/git
sudo chown git:git /srv/git
git init --bare /srv/git/project.git

Filebrowser Setup

  1. Install Filebrowser: 
    curl -fsSL https://raw.githubusercontent.com/filebrowser/get/master/get.sh | bash
  2. Create config database: filebrowser -d /etc/filebrowser.db config init
  3. Set up admin user: filebrowser -d /etc/filebrowser.db users add admin --password 'yourpassword'
  4. Configure as service: 
    [Unit]
Description=Filebrowser Service
After=network.target

[Service]
User=git
ExecStart=/usr/local/bin/filebrowser -d /etc/filebrowser.db serve
Restart=on-failure

[Install]
WantedBy=multi-user.target

Future Expansions

Media Server

Set up Plex or Jellyfin to stream your media collection across devices

sudo pacman -S jellyfin Default port: 8096

Music Streaming

Install Navidrome or Airsonic to access your music library anywhere

docker run -d navidrome/navidrome Default port: 4533

Cloud Storage

Deploy NextCloud for personal cloud storage and file sharing

sudo pacman -S nextcloud Default port: 8080

Thanks to Tailscale's mesh network, all these services will be securely accessible from any of your devices!

End Notes

  • Backup Strategy: Set up regular backups using rsync or restic
  • Monitoring: Install htop and iotop for resource monitoring
  • Security: Configure UFW firewall and fail2ban
  • Updates: Set up automatic security updates
  • Documentation: Keep a log of all configurations

Impact Analysis: Self-Hosted Linux Server

Sustainable Home Server Initiative

This project demonstrates how repurposed hardware can create an efficient, private, and environmentally conscious alternative to cloud services. By giving new life to old technology, we reduce e-waste while building a powerful self-hosted solution.

Environmental Impact

E-Waste Reduction

By repurposing an old laptop, this project prevents approximately 2.5kg of e-waste from reaching landfills, where it could have released harmful chemicals into the environment.

Manufacturing Impact

Reusing existing hardware eliminates the need for new server equipment, saving roughly 300-400kg of CO‚ÇÇ emissions that would have been generated in manufacturing.
